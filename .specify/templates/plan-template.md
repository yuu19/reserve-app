## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

- **I. Existing Architecture**: 既存の pnpm/Turborepo、Cloudflare Workers/Hono、
  SvelteKit、Expo、D1、Better Auth、Stripe/Resend/Sentry baseline を拡張しているか。
  置き換えアーキテクチャや無関係なリファクタを導入していないか。
- **II. Type Safety と API Boundary**: 外部入力、DTO、session、DB 由来値を
  `unknown`/type guard/Zod/domain union で絞り込み、`any` や未検証の型アサーションで
  問題を隠していないか。
- **III. Authorization と Scope**: organization/classroom scope、`facts -> effective ->
  sources -> display`、owner-only billing authority、billing responsibility と
  operational permission の分離を維持しているか。
- **IV. Risk-Based Verification**: 影響する risk surface に応じて、backend integration、
  web server、browser、mobile manual check、bug regression test を選定しているか。
- **V. Data, Billing, Deployment Safety**: D1、billing、entitlement、Stripe webhook、
  deployment に影響する場合、migration、既存データ互換性、冪等性、順不同耐性、
  audit/reconciliation signal、docs/README.md の deploy order を扱っているか。
- **VI. UI と Design System**: user-facing UI が DESIGN.md に従い、状態、操作、
  結果、不可理由を色だけに依存せず伝えているか。
